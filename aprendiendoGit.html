#1 Creando un repositorio en Git

Empiezo escribiendo los comando 'git init' y ya esta!!! lo que hace este comando es crearme una carpeta oculta en el directorio en el que voy a trabajar y automaticamente git ya esta trabanajando el el. Tambien guardo los cambios echos en los archivos escribiendo 'git add .' (enter) 'git commit -m('mensaje')'. git rm --cached: Elimina los archivos del área de Staging y del próximo commit pero los mantiene en nuestro disco duro. git rm --force: Elimina los archivos de Git y del disco duro. 'git reset HEAD': Este es el comando para sacar archivos del área de Staging. No para borrarlos ni nada de eso, solo para que los últimos cambios de estos archivos no se envíen al último commit, a menos que cambiemos de opinión y los incluyamos de nuevo en staging con git add.

#2 Definiendo ramas

Por defecto Git comienza con la rama master, trabajamos sobre esa rama al principio, pero si queremos probar cambios o trabajamos en equipo, es de buena practica generar cambios o trabajar en una rama que no sea la principal(master). Para eso creamos nuestra propia rama de trabajo con el comando 'git branch "nombre de la rama". Con 'git checkout "nombre de la rama" nos movemos hacia la rama nombrada y podemos empezar a trabajar en ella sin afectar los cambios en la rama principal. Por ultimo una vez tengamos los cambios echos en nuestra rama de prueba y echo el commit, podemos fucinar las ramas para que en nuuestra vesion principal se guarden los cambios, para eso tenemos que ir a la rama master y hacer un merge, 'git merge "nombre de la rama de prueba"'. Al hacer un merge tenemos que agregar un mesaje.

#3 Creando Tags

Los tags son una etiqueta de versiones de tu proyecto. Para ponerlas tenemos que escribir los comandos 'git tag -a "nombre de la version" -m "mensaje" "numero de commit al que esta asociado el tag". Para enviar un tag a Github git push origin(nombre del reppositorio remoto) --tags
